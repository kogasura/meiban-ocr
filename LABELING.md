# LABELING.md — Claude Code 用ラベリング指示書

このドキュメントは、`samples/` 内の画像から訓練データを生成するための
Claude Code 用の作業指示書です。

## このファイルの使い方

Claude Code セッションで以下のようにユーザーが指示します:

> `LABELING.md` を読んで、その手順に従って `samples/` 内の全画像をラベリングしてください。

Claude Code はこの指示書を読み、以下の手順を実行します。

---

## 作業の目的

`samples/` 内の銘板画像から、Ericsson シリアルコードとその位置情報を抽出し、
訓練データのアノテーションとして `annotations/` に保存します。

---

## 対象コード仕様

### Ericsson 銘板のシリアル

- **厳格パターン**: `/^E[39]\d{2}MM\d{6}$/`
- **部分一致パターン**: `/E[39]\d{2}MM\d{6}/`
- フォーマット: `E` + (`3` or `9`) + 数字2桁 + `MM` + 数字6桁
- 例: `E300MM000032`, `E300MM999022`

### 重要な観察ポイント

- 銘板は金属光沢の矩形ラベル
- 1画像に複数のラベルが写っていることが多い (グリッド配置)
- 「製造番号」のラベルが直前にあり、その右側にシリアルが印字されている

---

## 作業手順

### Step 1: 画像リストの取得

```bash
ls samples/*.{jpg,jpeg,png,JPG,JPEG,PNG} 2>/dev/null
```

すべての画像ファイルをリストアップしてください。

### Step 2: 各画像のラベリング

各画像について、以下を実行:

1. `view` ツールで画像を表示
2. 画像内のすべての Ericsson 銘板ラベルを特定
3. 各ラベルについて以下を判定:
   - **text**: 読み取ったシリアル文字列 (12文字)
   - **bbox**: ラベル全体 (金属プレート全体) の境界ボックス `[x1, y1, x2, y2]` (画像座標, ピクセル)
   - **text_bbox**: シリアル文字列**だけ**を囲む境界ボックス
     - テキスト書き換え augmentation で使う重要なフィールド
     - シリアル番号の文字列の上下左右をピッタリ囲む (余白最小)
   - **confidence**: 読み取りの自己評価 (0.0 〜 1.0)
   - **is_clear**: 文字が鮮明に読めるか (true / false)

4. 結果を `annotations/{basename}.json` に保存

### Step 3: スキーマ

```json
{
  "image": "samples/img_001.jpg",
  "image_size": [4000, 3000],
  "source_video": "videos/v1.mp4",
  "labels": [
    {
      "id": 0,
      "bbox": [120, 340, 480, 420],
      "text_bbox": [180, 360, 460, 410],
      "text": "E300MM000032",
      "confidence": 0.99,
      "is_clear": true
    },
    {
      "id": 1,
      "bbox": [520, 340, 880, 420],
      "text_bbox": [580, 360, 860, 410],
      "text": "E300MM000033",
      "confidence": 0.98,
      "is_clear": true
    }
  ]
}
```

**フィールド説明**:
- `bbox`: ラベル全体 (金属プレート全部) の境界ボックス
- `text_bbox`: シリアル文字列**だけ**の境界ボックス (テキスト書き換え augmentation 用、重要)
- `source_video`: 動画から抽出されたフレームの場合は元動画のパス (null可)

### Step 4: サマリレポート生成

全画像の処理が完了したら、`annotations/_report.md` に統計サマリを保存:

```markdown
# Labeling Report

- 処理画像数: N
- 検出ラベル総数: M
- 平均ラベル数/画像: M/N

## 信頼度分布
- confidence >= 0.95: X件 (XX%)
- 0.80 <= confidence < 0.95: Y件
- confidence < 0.80: Z件

## is_clear 分布
- true: A件
- false: B件 (要確認)

## 警告
- パターンに一致しないシリアル: (リスト)
- bbox が画像外に出ている: (リスト)
- 重複しているシリアル: (リスト)

## サンプル別ラベル数 (上位/下位)
- img_005.jpg: 24 labels
- img_001.jpg: 20 labels
- ...
```

## 重要な指針

### bbox の決め方

- ラベルの**金属部分全体**を含む矩形を取る
- ラベル外の背景は最小限に
- 余白の目安: ラベル端から 2〜5px 程度
- 完全なピクセル精度は不要 (後で OpenCV で refine する)

### confidence の付け方

- **0.95 以上**: 全文字が明瞭に読める、誤読の可能性なし
- **0.80〜0.95**: ほぼ確実だが一部の文字に微妙な不確かさ
- **0.50〜0.80**: 推測を含む読み取り
- **0.50 未満**: 読めない or 大きな推測

### is_clear の判定

- 全12文字が明瞭で、紛らわしい文字 (O/0, I/1 等) の判別も明確 → `true`
- ブレ・反射・部分欠けで一部文字が不明瞭 → `false`
- `false` のサンプルは訓練から除外されるが、test セットとして重要

### パターン違反への対応

検出したテキストが Ericsson の厳格パターン `/^E[39]\d{2}MM\d{6}$/` に一致しない場合:

1. **読み取り誤りの可能性が高い** → confidence を下げる、is_clear を false に
2. ただしテキスト自体は推測したものを記録する (補正なし、生の読み取りを残す)
3. `_report.md` の警告セクションに記録

### スキップの判断

以下の画像は処理対象外として `annotations/_skipped.json` に理由付きで記録:

- 画像が壊れている / 読み込めない
- 銘板が1つも写っていない
- 完全にブレていて読み取り不可

50枚処理ごとに簡潔な進捗報告をしてください:

> 50/100 完了。検出ラベル総数: 950件、平均 confidence: 0.96。
> パターン違反: 2件。続行します。

全件完了後、`annotations/_report.md` を生成し、ユーザーに完了報告。

## トラブルシューティング

### bbox がずれる気がする

VLM の bbox 精度は ±10〜30px 程度なので、ピクセル精度は不要。
後段の `refine_bbox.py` (OpenCV) で精密化されるため、概略でOK。

### 画像が大きすぎる

そのままの座標で記録してください。リサイズ等は後段で行います。

### 画像内のラベルが20件超ある

すべて検出してください。1JSONあたりのラベル数に上限はありません。

### `MM` 以外のフォーマットが見える (`MR`, `MN` 等)

Ericsson の正規パターンに違反するため、誤読の可能性が高い。
推測した文字を記録した上で、is_clear を false にしてください。

---

## 注意事項

- このタスクは**1〜2時間程度の長時間セッション**になります。途中で進捗を区切ってください
- 同じ画像を2回処理しないよう、既に `annotations/{basename}.json` が存在する画像はスキップ
- `samples/` を変更しないこと (読み取り専用)
- `annotations/` 以下のみ書き込み可
